For pro and con of using server-side rendering. The first advantage is it lets the user have a better experience. Since it lets the page loads faster and make it to be more efficient. Second, it makes user to be more convenient. Since the content will be stored before loading the page, the search engine can easily draw those content. On the other hand, the first disadvantage is it will be costly as the server-side have to store lots of data which means the server must have huge capacity of those data. Second, server-side is not able to use JavaScript coding. In other words, it can just use for simple pages. Otherwise, the page rednering will become slower.

For pro and con of using client-side rendering. The first advantage is it is cheap and easy to host. Since web server is not required for client-server, it will be free for hosting it. Second, it does not require any full-page reload. In other words, users do not have to wait for the server response anymore. On the other hand, the first disadvantage is it will let users with slower devices have a bad experience. Since, by using their devices, it takes time to load and may let them not happy with it. Second, it will load slower than server-side. Since, an extra round-trip to API server is required, the time for loading will be increased.

For creating a page of allowing thousand of daily users, first thing need to be done is to plan the layer of the web and database. Also, a cache like CDN for storing temporary data is required. In this way, it avoids the round trip between the user and the database. Then, in order to allow user can access any web server at anytime, a stateless server is required. In this case, user's data will be stored in persistant database, but not inside the web server. Since there will be a huge amount of data, one server won't be able to store all of them. Therefore, those data will be sharded to several servers. Each data of each shard is unique.
Representational State Transfer is an architectural style for developing websites. In REST Architecture, it allows user to cache data and decide when to use response caching. Also, REST Architecture has a stateless HTTP protocol foundation, which means each request of the user has been isolated. No preserved information will be allowed. Furthermore, the REST Architecture always allow to create a multiple layers of servers. Therefore, REST would enable scaling for thousand of daily users.

For the benefits of cookies, first one will be it's avalibality. Once cookies have been stored, it will be avalable a long time until user deletes it manully. It can be retrieved even though the server is not working. Second, cookies are convenient. Cookies can be used in filling form. When user visit an address site, the information can be filled by cookies automatically. Third, cookies reduce the load of server. Since cookies are stored in the hard drive, not in the server. there won't be a burden issue for running it.
For the disadvantages of cookies, first one will be browser impact. Since cookies won't be restricted, it will be accumulated and the server will be slow down eventually unless user deletes it manully. Second, cookies cause security issue. Since every hacker can view the information from cookies, the user's information may be used in illegal way by the hacker. Third, cookies cause privacy issue. Since cookies will record all the online activities and user won't be aware, those record can be collected by government or other business company.

There will be three tests for my application including functional test, performance test and security test. For functinal test, it validates the software system against the functional requirements. The purpose of it is to test each function from the page, by providing input, checking the output agaist the functional requirements. It is important that to make sure the function can run properly. Otherwise, the whole page cannot be shown due to error. The example of functional test including unit testing, smoke testing and sanity testing.
The performance test determines how the stability, speed, scalability, and responsiveness of the page works under a given workload. Since the application should handle a thousand of users, performance test need to be used to test whether it fulfils the performance reqirements and the stability under peak traffic events. Before executing the performance test, the test environment and tools should be prepared which is depended on each organisation.
The security test is to identify threats of the page and measure the potential risks of it, and provide ways to encounter those risks. Since the page will stored lots of user information, security test is important to ensure the user's security. For running the security test, a test plan should be required which is including security-related tests, test data, test tools and the analysis of various tests outputs from different security tools.
